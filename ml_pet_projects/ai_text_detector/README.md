# AI Text Detector

Pet project based on `task7.ipynb`.

The project detects whether a text was written by a human or generated by a
machine. The notebook fine-tuned `bert-base-uncased` with frozen lower layers;
this repository version presents it as a reproducible ML project with a fast
TF-IDF baseline for local checks and a documented BERT fine-tuning setup for the
full experiment.

## Why it is portfolio-ready

- Clear binary NLP task with an interpretable baseline.
- BERT architecture path from the original notebook: frozen embeddings and lower
  encoder layers, trainable upper layers, dropout classifier head.
- Balanced data preparation matching the HuggingFace dataset label convention:
  `0 = Human`, `1 = Machine`.
- Metrics and reports saved as GitHub-friendly artifacts.

## Full-data direction

The original notebook uses the HuggingFace dataset
`andythetechnerd03/AI-human-text`, tokenizes texts with `bert-base-uncased`,
freezes the first 10 BERT layers, and trains a binary classifier for 3 epochs.
That path is intentionally kept out of smoke tests because it downloads a model
checkpoint and is much better with GPU.

The CLI accepts any CSV with columns:

- `text`
- `label` or `generated`, where `0 = Human`, `1 = Machine`

```bash
PYTHONPATH=src python -m ai_text_detector.cli --data data/ai_human_text_sample.csv
```

Artifacts are saved to `outputs/ai_text_detector`:

- `metrics.csv`
- `bert_setup.csv`
- `classification_report.txt`
